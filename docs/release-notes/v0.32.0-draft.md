# v0.32.0 — ponytail-audit over-engineering 정리 arc

"1인용 RAG 치고 너무 복잡해졌다"는 문제의식에서 출발한 over-engineering 감사(ponytail-audit) 결과를 네 PR(#219–#222)로 정리한 릴리스. **능력(검색·청킹·RAG 품질)은 전부 유지하되 조작 표면·코드 구조·crate 수를 단순화.** 누적 약 **−3200줄, crate 24→20, dep −2(serde_yaml + transitive unsafe-libyaml)**.

`minor` bump 트리거는 #219 의 CLI 플래그/config 키 제거(인터페이스 변경) 하나뿐 — 나머지 셋은 출력·동작 byte-identical 이라 자체로는 patch 급. 실엔진 도그푸딩 evidence 는 `tasks/HOTFIXES.md` 2026-06-27 항목.

## 1. 죽은 search-cache scaffold 제거 (#219) — minor (인터페이스 변경)

in-process LRU search cache 는 spine 재작성(#214, v0.31.0)에서 이미 삭제됐는데, 그 비계만 코드·CLI·문서에 남아 있었다. 잔존물을 걷어낸다.

- **변경 사실**: `kebab search --no-cache` 와 `kebab search --explain` 플래그 제거(둘 다 이미 no-op/미배선이었음). `[rag] explain_default` config 키 제거. 내부적으로 `App::search`/`search_uncached` 붕괴 + `search_uncached_with_config` facade 삭제.
- **trade-off**: `kebab search --no-cache`/`--explain` 를 타이핑하던 muscle memory 가 있으면 "unexpected argument" 에러가 난다. (실제로 둘 다 아무 동작도 안 하던 죽은 플래그였다.) `kebab ask --explain` 은 **그대로 live** — 영향 없음.
- **mitigation**: search 의 ranking 디테일이 필요하면 `kebab search --trace`(live)가 pre-fusion lexical/vector/RRF 후보 + per-stage timing 을 보여준다. `[rag] explain_default` 가 든 옛 config.toml 은 그대로 로드된다(serde 가 unknown 키 무시 — 마이그레이션 불필요).
- **upgrade**: 자동. 새 바이너리부터 두 플래그가 사라진다. `search_cache: false` wire capability(`schema.v1`)는 정직하게 유지(없는 캐시를 false 로 보고).

## 2. 9개 동일 code AST chunker → CodeAstV1Chunker 통합 (#220) — 출력 byte-identical

`code_{c,cpp,go,java,js,kotlin,python,rust,ts}_ast_v1.rs` 9개는 **struct 이름 + `VERSION_LABEL` 문자열만 다른 byte-identical 복붙**이었다(chunker 는 tree-sitter 미사용, `lang` 은 `SourceSpan::Code` 데이터에서 흐름). 단일 `CodeAstV1Chunker { version_label }` + `for_lang(lang)` 로 통합(**−3030줄**).

- **변경 사실**: 9개 src 파일 → 1개. `for_lang` 이 9개 언어를 기존 `VERSION_LABEL` 문자열로 **verbatim** 매핑(`rust→code-rust-ast-v1`, `typescript→code-ts-ast-v1` 등).
- **trade-off**: 없음 — `chunker_version` 문자열이 언어별로 동일하게 보존돼 `chunk_id`(= `id_for_chunk(doc, chunker_version, …)`)가 byte-identical → **재인덱싱 0, 임베딩 드리프트 0**.
- **mitigation**: 9개 골든 스냅샷 테스트가 expected 무수정 통과(단위 증명) + **실코퍼스 A/B 도그푸딩**: pre-arc(9-chunker) vs post-arc(consolidated) 바이너리로 같은 코드 코퍼스 ingest → 120 chunks 동일, chunk_id+chunker_version diff **완전 비어있음**.
- **upgrade**: 자동. 재인덱싱 불필요(같은 chunk_id 라 idempotent upsert 가 no-op).

## 3. 빈 re-export shim crate 흡수 (#221) — 동작 불변 (crate 22→20)

`kebab-embed`/`kebab-llm` 은 "새 type 없음"을 자처한 순수 re-export 셸이었다(trait 은 이미 `kebab-core` 소유, mock + test helper 만 보유). `kebab-core` 의 default-OFF `mock` feature 로 흡수.

- **변경 사실**: 두 crate 삭제, `MockEmbedder`/`MockLanguageModel`/`assert_*` helper 를 `kebab-core` 로 이동(`mock` feature, dev-dep 에서만 활성). workspace crate 22 → 20.
- **trade-off**: 없음 — production 빌드는 mock 모듈을 컴파일하지 않음(default-OFF). trait surface·동작 불변. 외부 integration 영향 0(단일 바이너리).
- **mitigation**: mock byte-identical(import rewrite 만), 의존 crate 테스트 전부 녹색.
- **upgrade**: 자동. 사용자 표면 무변경.

## 4. FusionPolicy inline + dual-YAML 통일 + NLI default 제거 (#222) — 출력 불변

audit 잔여 작은 정리 셋. (1) 변종 하나뿐이던 `FusionPolicy::Rrf` enum 을 `HybridRetriever.k_rrf: u32` 로 inline(RRF 수식 byte-identical). (2) `serde_yaml`(0.9, archived) → `serde_yaml_ng`(0.10, maintained fork) 일원화 — transitive `unsafe-libyaml` 까지 제거. (3) `NliVerifier::hypothesis_token_count` 의 `Ok(0)` default 제거(prod 가 이미 override 라 동작 불변, footgun 을 compile-time 보장으로).

- **trade-off**: 없음 — 검색 순위·YAML 파싱·prod NLI 동작 전부 불변.
- **upgrade**: 자동.
